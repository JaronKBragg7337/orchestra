# Cycle manifests

Each manifest is the mechanical status record for one collection cycle. It records every source × domain capture, including retries, refusals, model changes, unavailable sources, and unknown metadata.

Manifests report capture completeness and content substance separately. They do not interpret what a response means.

For multi-attempt captures, manifests also distinguish unavailable attempts from the final content result. Refusal counts describe captured model responses; availability counts describe provider or account conditions.
